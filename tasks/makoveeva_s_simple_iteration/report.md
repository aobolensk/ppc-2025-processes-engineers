## Постановка задачи
Требуется реализовать метод простой итерации для решения системы линейных уравнений:

последовательную версию алгоритма;

параллельную версию с использованием MPI;

Для обеих реализаций необходимо:

обеспечить корректность вычислений;

реализовать этапы валидации, предобработки, вычислений и постобработки;

провести функциональное и производительное тестирование.

## Описание алгоритма
Метод простой итерации основан на последовательном уточнении приближённого решения системы линейных уравнений.

Алгоритм работы метода следующий:

Формируется матрица коэффициентов и вектор правой части.

Начальное приближение вектора решения задаётся нулевым.

На каждой итерации:

для каждой строки матрицы вычисляется новое значение соответствующего элемента вектора решения;

используется параметр релаксации для повышения устойчивости и сходимости метода;

После каждой итерации вычисляется величина изменения решения.

Если изменение становится меньше заданного порога точности, итерационный процесс завершается.

В качестве результата вычисляется сумма элементов полученного вектора решения.

## Особенности реализации
Последовательная версия (SEQ)
Все вычисления выполняются в одном потоке.

Матрица и вектор формируются локально.

Итерационный процесс выполняется в одном цикле.

Проверка сходимости производится после каждой итерации.

Используется стандартный таймер на основе std::chrono.

Последовательная версия служит базовой реализацией для проверки корректности и сравнения производительности.

Параллельная версия (MPI)
Матрица распределяется по строкам между MPI-процессами.

Каждый процесс вычисляет локальную часть нового вектора решения.

Для обмена данными используются коллективные операции MPI:

распределение данных (MPI_Scatterv);

сбор обновлённых частей решения (MPI_Allgatherv);

вычисление глобальной нормы ошибки (MPI_Allreduce);

Проверка сходимости производится по глобальному значению ошибки.

Итоговый результат формируется на процессе с рангом 0.

## Анализ результатов
Параллельная MPI-реализация демонстрирует снижение времени выполнения при увеличении числа процессов до двух.

При дальнейшем увеличении числа процессов влияние накладных расходов на коммуникации возрастает.

Последовательная версия работает медленнее по сравнению с MPI при использовании нескольких процессов.

При одном процессе время выполнения SEQ и MPI-версий сопоставимо.

Все реализации корректно проходят функциональные и интеграционные тесты.
